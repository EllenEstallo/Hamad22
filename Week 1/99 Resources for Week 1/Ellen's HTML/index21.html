How Forms Work

  1. A user fills in a form then presses a button to submit the info to server.
  2. The NAME of each FORM CONTROL is sent to the server along with the VALUE the user enters or selects.
  3. The server processes the info, & may store it in a database.
  4. Server creates a new page to sent back to browser based on info received.

Note: A form may have several form controls, each gathering different information. Name=Value

Form Structure

FORM

  1. Form controls live inside a 'form' element.
  2. This element should always carry the action attribute and will usually have a method and id attribute too.

ACTION

  1. Every 'form' element requires an action attribute.
  2. Its value is in the URL fo the page on the server that will receive the information in the form when it is submitted.

METHOD

Forms can be sent using one of two methods: get or post.

  1. With the get method, the values from the form are added to the end of the URL specified in the action attribute. The get method is ideal for:
    • short forms/search boxes
    • when you are just retrieving data from the web server (not sending information that should be added to or deleted from a database.)
  2. With the post method, the values are sent in what are known as HTPP headers. You should use the post method if your form:
    • allows users to upload a Files
    • is very long
    • constains sensitive data (passwords, etc)
    • adds info to, or deletes info from a database

Note: If the method attribute is not used, the form data will be sent using the get method.
